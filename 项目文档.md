## 1. 项目概述

### 背景

学生日常学习任务（课程作业、实验报告、考试安排等）分散在笔记本、便签、聊天记录中，容易遗漏。截止时间临近时缺乏有效提醒，逾期后才发现未提交。

### 目标

构建一个轻量级 Web 应用，提供任务创建、优先级设置、截止时间提醒、分类整理、日历视图等功能，将零散的学习任务集中管理，提高学习安排的系统性和执行效率。

### 适用场景

- 大学生管理多门课程的作业与实验
- 跟踪考试复习进度
- 规划日常学习日程
- 避免遗漏截止日期

## 3. 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| **后端框架** | FastAPI | 0.115.6 |
| **数据校验** | Pydantic | 2.10.3 |
| **服务器** | Uvicorn | 0.34.0 |
| **数据库** | SQLite | 内置 |
| **前端** | HTML5 + CSS3 + Vanilla JS | — |
| **Python** | 3.13+ | — |

### 选型理由

- **FastAPI**：异步高性能，自动生成 OpenAPI 文档，类型安全
- **SQLite**：零配置、免安装、单文件存储，适合轻量级应用
- **原生前端**：无框架依赖，加载快，部署简单，适合单页应用场景
- **Uvicorn**：ASGI 服务器，生产级性能

## 6. 环境变量

| `TASK_APP_HOST` | `0.0.0.0` | 服务监听地址 |

```bash
# Linux / macOS
export TASK_APP_PORT=9000
export TASK_REMIND_HOURS=48
python -m uvicorn main:app --host 0.0.0.0 --port $TASK_APP_PORT
```

## 7. API 接口文档

### 基础信息

- **Base URL**: `http://127.0.0.1:8080/api`
- **Content-Type**: `application/json`
- **OpenAPI 文档**: 启动服务后访问 `http://127.0.0.1:8080/docs`

---

### 7.1 获取任务列表

```
GET /api/tasks
```

**查询参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `status` | string | 否 | 按状态筛选：`未开始` / `进行中` / `已完成` / `已逾期` |
| `course` | string | 否 | 按课程名称筛选 |
| `task_type` | string | 否 | 按类型筛选：`作业` / `考试` / `实验` / `其他` |
| `priority` | string | 否 | 按优先级筛选：`高` / `中` / `低` |
| `search` | string | 否 | 模糊搜索标题和描述 |

**请求示例：**

```bash
# 获取全部任务
curl http://127.0.0.1:8080/api/tasks

# 筛选：进行中的作业
curl "http://127.0.0.1:8080/api/tasks?status=进行中&task_type=作业"

# 搜索
curl "http://127.0.0.1:8080/api/tasks?search=高数"
```

**响应示例：**

```json
{
  "count": 2,
  "tasks": [
    {
      "id": 1,
      "title": "完成高数作业第五章",
      "description": "1-10题，注意证明题步骤",
      "course_name": "高等数学",
      "task_type": "作业",
      "priority": "高",
      "deadline": "2026-06-18T15:00",
      "status": "进行中",
      "created_at": "2026-06-16 18:11:47",
      "updated_at": "2026-06-16 18:11:47"
    }
  ]
}
```

### 7.3 创建任务

```
POST /api/tasks
```

**请求体：**

```json
{
  "title": "完成高数作业",
  "description": "第五章 1-10题",
  "course_name": "高等数学",
  "task_type": "作业",
  "priority": "高",
  "deadline": "2026-06-18T15:00",
  "status": "未开始"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` | string | **是** | 任务标题 |
| `description` | string | 否 | 任务描述，默认 `""` |
| `course_name` | string | 否 | 课程名称，默认 `""` |
| `task_type` | string | 否 | 任务类型，默认 `"其他"` |
| `priority` | string | 否 | 优先级，默认 `"中"` |
| `deadline` | string | **是** | 截止时间，ISO 格式 |
| `status` | string | 否 | 初始状态，默认 `"未开始"` |

**响应：** `201 Created`

```json
{ "id": 2, "message": "任务创建成功" }
```

## 9. 数据库设计

### 表：`tasks`

```sql
CREATE TABLE tasks (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    title       TEXT    NOT NULL,              -- 任务标题
    description TEXT    DEFAULT '',            -- 任务描述
    course_name TEXT    DEFAULT '',            -- 课程名称
    task_type   TEXT    DEFAULT '其他',        -- 类型：作业/考试/实验/其他
    priority    TEXT    DEFAULT '中',          -- 优先级：高/中/低
    deadline    TEXT    NOT NULL,              -- 截止时间 ISO 格式
    status      TEXT    DEFAULT '未开始',      -- 状态：未开始/进行中/已完成/已逾期
    created_at  TEXT    DEFAULT (datetime('now','localtime')),
    updated_at  TEXT    DEFAULT (datetime('now','localtime'))
);
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PK, 自增 | 唯一标识 |
| `title` | TEXT | NOT NULL | 任务标题 |
| `description` | TEXT | — | 详细描述 |
| `course_name` | TEXT | — | 所属课程 |
| `task_type` | TEXT | — | 作业 / 考试 / 实验 / 其他 |
| `priority` | TEXT | — | 高 / 中 / 低 |
| `deadline` | TEXT | NOT NULL | ISO 8601 格式 |
| `status` | TEXT | — | 未开始 / 进行中 / 已完成 / 已逾期 |
| `created_at` | TEXT | — | 创建时间 |
| `updated_at` | TEXT | — | 最后更新时间 |

### 索引策略

当前数据量较小，未添加额外索引。若任务数超过 1000 条，建议：

```sql
CREATE INDEX idx_tasks_status ON tasks(status);
CREATE INDEX idx_tasks_deadline ON tasks(deadline);
CREATE INDEX idx_tasks_course ON tasks(course_name);
```

## 附录

### A. 常见问题

**Q: 如何备份数据？**

```bash
# 复制数据库文件即可
cp student_tasks.db student_tasks_backup.db
```

**Q: 如何修改提醒时间？**

```bash
# 改为 48 小时
set TASK_REMIND_HOURS=48
```

**Q: 如何重置所有数据？**

```bash
# 删除数据库文件，重启服务会自动创建空表
rm student_tasks.db
```

**Q: 如何查看 API 文档？**

启动服务后访问：http://127.0.0.1:8080/docs （Swagger UI）或 http://127.0.0.1:8080/redoc （ReDoc）
